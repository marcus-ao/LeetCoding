# LeetCode题目分析Skill

## Skill概述

你现在需要使用位于 `leetcode-analyze-skill/` 目录下的LeetCode题目分析Skill。这个Skill专门用于系统化地分析LeetCode算法题目，提供从题目理解到代码实现的完整解题思路。

## Skill文件结构

```
leetcode-analyze-skill/
├── SKILL.md                          # Skill主文档（核心工作流程和规范）
├── assets/
│   └── analysis-template.md          # 输出文档模板
└── references/
    ├── algorithm-patterns.md         # 常见算法模式参考（10个核心模式）
    ├── analogy-examples.md           # 形象化类比示例库
    ├── code-filling-guide.md         # 代码填空学习法指南
    ├── complexity-guide.md           # 复杂度分析指南
    └── problem-categories.md         # 题目分类和相似题目库
```

## 使用流程

### 第1步：识别Skill调用
当用户的请求包含以下特征时，你应该使用这个Skill：
- 用户明确提到"LeetCode"、"算法题"、"题目分析"、"解题思路"
- 用户提供了题目文件或题目描述
- 用户询问算法优化、复杂度分析、代码实现
- 用户需要从暴力解法到优化解法的推导过程
- 用户提到"易错点"、"相似题目"、"代码填空"

### 第2步：读取Skill主文档
**立即读取** `leetcode-analyze-skill/SKILL.md` 文件，这是Skill的核心文档，包含：
- 完整的工作流程（5个阶段）
- 每个阶段的详细要求
- 输出规范和格式要求
- 教学原则和代码实现要求

**关键要求**：
1. 必须严格按照SKILL.md中定义的5个阶段进行分析
2. 必须使用 `assets/analysis-template.md` 作为输出模板
3. 必须在适当时机查阅references目录下的参考文档

### 第3步：读取输出模板
**读取** `leetcode-analyze-skill/assets/analysis-template.md`，这是生成分析文档的标准模板，包含：
- 完整的章节结构（10个章节）
- 每个章节的具体格式要求
- 填空练习的设计模板
- 答案解析的呈现方式

**必须遵守**：
- 输出文档必须包含模板中的所有章节
- 章节顺序和格式必须与模板一致
- 使用模板中定义的emoji和Markdown格式

### 第4步：执行5个阶段的分析

#### 阶段1：题目理解
1. 读取用户提供的题目文件或描述
2. 提取关键信息：编号、标题、描述、示例、约束条件
3. 用自己的话重新描述问题
4. 识别题目类型
5. 提取关键观察点和突破口

#### 阶段2：解题思路分析

**前置：形象化类比讲解**
- **必须先查阅** `leetcode-analyze-skill/references/analogy-examples.md`
- 查找是否有适合当前题目的现成类比
- 如果有，使用或调整该类比
- 如果没有，按照文档中的"Agent自主类比创作指南"（第6节）自行创作
- 类比必须在讲解暴力解法或优化解法之前提供

**方法一：暴力解法**
1. 如果暴力解法涉及复杂概念，先提供形象化理解
2. 描述最直观的解决方案
3. 分析复杂度（查阅 `references/complexity-guide.md`）
4. 指出瓶颈

**方法二：优化解法**
1. **必须提供形象化理解**（重点解释优化的核心思想）
2. 从暴力解法的瓶颈出发，逐步推导优化思路
3. **查阅** `leetcode-analyze-skill/references/algorithm-patterns.md` 寻找适用的算法模式
4. **重要**：不要局限于这10个模式，如果题目需要其他方法，应该独立思考并创新
5. 详细描述优化解法的算法步骤
6. 分析复杂度（查阅 `references/complexity-guide.md`）

#### 阶段3：图解说明
1. 使用Mermaid图表或清晰的文字描述展示关键步骤
2. 可视化数据结构的变化过程
3. **注意**：Mermaid图表中避免在方括号内使用双引号和括号

#### 阶段4：代码实现

**关键要求**：填空练习的代码必须与完整实现的代码完全一致！

**步骤1：代码框架填空**
1. **查阅** `leetcode-analyze-skill/references/code-filling-guide.md` 了解填空设计原则
2. 使用ACM模式提供代码骨架
3. 关键代码挖空（用 `______` 标记）
4. 为每个填空提供思考提示（不直接给答案）
5. 根据难度分级设计填空（简单/中等/困难）

**步骤2：完整代码实现**
1. **Python实现（主要）**：
   - 使用ACM模式
   - 提供完整函数实现
   - 添加详细行内注释
   - 说明关键步骤和技巧
   - **对比填空答案，解释为什么这样填**

2. **C++实现（次要）**：
   - 使用ACM模式
   - 提供完整函数实现
   - 说明与Python的语法差异

#### 阶段5：补充信息

**易错点提醒**
1. 列出常见边界条件
2. 指出容易犯的错误
3. 提供调试和验证技巧

**相似题目推荐**
1. **查阅** `leetcode-analyze-skill/references/problem-categories.md`
2. 推荐2-3道同类型按难度递增的题目
3. 建议学习路径
4. 总结相关知识点

### 第5步：生成输出文档

**文档命名**：`{题目编号}-{题目名称}-思路分析.md`
- 示例：`1-TwoSum-思路分析.md`
- 示例：`20-ValidParentheses-思路分析.md`

**文档内容**：
- 严格按照 `assets/analysis-template.md` 的结构
- 包含所有10个章节
- 使用正确的Markdown格式和emoji
- 确保代码可以直接运行

## 参考文档使用指南

### 何时查阅各个参考文档

| 文档 | 何时查阅 | 用途 |
|------|---------|------|
| `algorithm-patterns.md` | 阶段2：分析优化解法时 | 查找适用的算法模式（10个核心模式） |
| `analogy-examples.md` | 阶段2：讲解思路前 | 查找或创作形象化类比 |
| `code-filling-guide.md` | 阶段4：设计填空练习时 | 了解填空设计原则和难度分级 |
| `complexity-guide.md` | 阶段2：分析复杂度时 | 准确分析时间和空间复杂度 |
| `problem-categories.md` | 阶段5：推荐相似题目时 | 查找同类型题目和学习路径 |

### 重要原则

1. **algorithm-patterns.md 的使用**：
   - 这个文档只提供10个最常用的核心算法模式
   - **不要局限于这些模式**
   - 如果题目需要其他算法或创新思路，应该独立思考
   - 算法模式是参考工具，不是唯一答案

2. **analogy-examples.md 的使用**：
   - 优先查找现成的类比
   - 如果没有合适的，按照文档第6节"Agent自主类比创作指南"自行创作
   - 类比必须：具体可感、本质相关、易于理解、可扩展

3. **code-filling-guide.md 的使用**：
   - 理解填空的教育价值
   - 掌握"保留什么"和"挖空什么"的原则
   - 根据题目难度设计合适的填空难度
   - 提供引导性提示而不是直接答案

## 常见错误和避免方法

❌ **错误1**：没有查阅参考文档就直接开始写
✅ **正确**：按照流程，在适当时机查阅对应的参考文档

❌ **错误2**：填空练习的代码与完整实现不一致
✅ **正确**：先设计完整实现，再从中挖空，确保一致性

❌ **错误3**：没有提供形象化类比就直接讲算法
✅ **正确**：在讲解复杂概念前，先查阅或创作生活化类比

❌ **错误4**：局限于algorithm-patterns.md中的10个模式
✅ **正确**：参考模式但不局限，根据题目特点独立思考

❌ **错误5**：输出文档缺少某些章节或格式不对
✅ **正确**：严格按照analysis-template.md的结构输出
