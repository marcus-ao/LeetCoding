# LeetCode Analyze Skill

## 概述

本Skill专门用于帮助用户系统化地分析LeetCode算法题目，提供从题目理解到代码实现的完整解题思路，特别适合算法初学者和准备技术面试的工程师。

## 工作流程

按照以下5个阶段进行题目分析：

### 阶段1：题目理解

1. 读取用户提供的题目Markdown文件
2. 提取关键信息（若题目描述不具有下列格式化的形式，则自行进行分析归纳整理相关信息）：
   - 题目编号和标题
   - 问题描述
   - 输入输出示例
   - 约束条件
3. 用自己的话重新描述问题，确保理解正确
4. 识别题目类型（数组、链表、树、图、动态规划等）
5. 提取关键观察点和突破口

### 阶段2：解题思路分析

#### 前置：形象化类比讲解（降低理解难度）

**目的**：在进入专业算法分析之前，先用生活化的例子帮助理解核心概念

1. **识别难点**：判断解题思路中可能让初学者困惑的概念
   - 抽象的数据结构（如哈希表、栈、队列等）
   - 复杂的算法思想（如动态规划、回溯、贪心等）
   - 不直观的操作过程（如双指针移动、滑动窗口等）

2. **设计类比**：为每个难点设计贴近生活的类比
   - 使用日常场景（购物、排队、找东西等）
   - 具体化抽象概念
   - 可视化操作过程

3. **类比讲解原则**：
   - **具体化**：用具体事物代替抽象概念
   - **可视化**：描述可以想象的画面
   - **关联性**：类比要与算法本质相关
   - **渐进性**：从类比过渡到算法概念

4. **类比示例模板**：
   ```
   🌟 形象化理解：[概念名称]
   
   想象这样一个场景：[生活化场景描述]
   
   在这个场景中：
   - [算法元素1] 就像 [生活中的对应物]
   - [算法操作1] 就像 [生活中的对应动作]
   
   通过这个例子，我们可以理解：[核心思想]
   
   现在让我们把这个思想应用到算法中...
   ```

5. **查阅参考**：使用 [`references/analogy-examples.md`](references/analogy-examples.md) 获取常见算法的类比示例

#### 方法一：暴力解法

1. **形象化理解**（如果暴力解法本身较复杂）
2. 从零开始一步一步地描述最直观的解决方案
3. 分析时间复杂度和空间复杂度
4. 指出暴力解法的瓶颈和为什么需要优化

#### 方法二：优化解法

1. **形象化理解**（重点：优化思路的核心概念）
2. 从暴力解法的瓶颈出发，一步一步给出思考优化方向
3. 查阅 [`references/algorithm-patterns.md`](references/algorithm-patterns.md) 寻找适用的算法模式
4. 逐步推导优化思路（如引入哈希表、双指针、滑动窗口等）
5. 详细描述最优解的算法步骤
6. 使用 [`references/complexity-guide.md`](references/complexity-guide.md) 进行复杂度分析

### 阶段3：图解说明

1. 使用Mermaid图表或清晰的文字描述展示关键步骤（流程图图形较少可使用横向展示，图形较多请使用竖向展示）
2. 可视化数据结构的变化过程
3. 展示算法执行的关键节点

**注意**：Mermaid图表中避免在方括号内使用双引号和括号，保持语法正确性与简洁性。

### 阶段4：代码实现（请务必保证步骤1挖空练习中的代码与步骤2完整实现中的代码保持一致，否则将极大影响学习一致的体验感）

#### 步骤1：代码框架填空（互动学习）

**目的**：帮助用户从"不知道怎么开始"过渡到"能够独立思考关键逻辑"

1. **使用ACM模式提供代码骨架**：给出函数签名、基本结构和注释框架，其中开头的库导入/引用等完整代码直接给出，但需要使用注释指明该库与余下代码中的哪些关键部分关联
2. **关键代码挖空**：将核心逻辑部分用 `______` 或 `# TODO: 填空` 标记
3. **填空提示**：为每个挖空处提供思考提示，引导用户思考
4. **难度分级**：
   - 简单填空：变量初始化、返回语句
   - 中等填空：循环条件、判断条件
   - 困难填空：核心算法逻辑

**填空设计原则**：
- 保留框架代码（库导入/引用、函数定义、基本结构）
- 挖空关键逻辑（算法核心、重要判断）
- 提供思考提示（不直接给答案，引导思考方向）
- 循序渐进（从简单到复杂）

#### 步骤2：完整代码实现

**Python实现（主要）**
1. 使用ACM模式实现完整代码
2. 提供完整的函数实现
3. 添加详细的行内注释
4. 说明关键步骤和技巧
5. 对比填空答案，解释为什么这样写

**C++实现（次要）**
1. 使用ACM模式实现完整代码
2. 提供完整的函数实现
3. 说明与Python的语法差异
4. 注意C++特有的内存管理和STL使用

### 阶段5：补充信息

**易错点提醒**
1. 列出常见的边界条件（空数组、单元素、重复元素等）
2. 指出容易犯的错误
3. 提供调试和验证技巧

**相似题目推荐**
1. 查阅 [`references/problem-categories.md`](references/problem-categories.md) 
2. 推荐2-3道同类型按难度递增的题目
3. 建议难度递进的学习路径
4. 总结相关知识点与核心难点

## 输出规范

### 文档命名

生成的分析文档命名格式：`{题目编号}-{题目名称}-思路分析.md`

示例：`1-TwoSum-思路分析.md`、`20-ValidParentheses-思路分析.md`、`2073-TimeNeededToBuyTickets-思路分析.md`

### 文档结构

使用 [`assets/analysis-template.md`](assets/analysis-template.md) 作为输出模板，包含以下章节：

1. **📋 题目信息** - 难度、标签、来源
2. **📖 题目描述** - 原题描述、示例、约束条件
3. **🤔 题目分析** - 从0开始的一步一步的问题思路理解、关键要点观察以及信息关联至核心数据结构选用的分析
4. **💡 解题思路** - 暴力解法和优化解法的完整推导
5. **🎨 图解说明** - 可视化关键步骤
6. **✏️ 代码框架填空** - 互动式学习，关键代码挖空
7. **💻 完整代码实现** - Python和C++完整实现
8. **⚠️ 易错点提醒** - 边界条件、常见错误、调试技巧
9. **🔗 相似题目推荐** - 同类型题目、进阶题目、相关知识点
10. **📚 知识点总结** - 核心算法和数据结构知识点

### 格式要求

- 使用清晰的Markdown格式
- 适当使用emoji增强可读性
- 代码块标注语言类型
- 复杂度分析使用Big O表示法
- 保持专业和教学导向的语气

## 代码实现要求

1. **Python为主**：提供详细的Python实现，这是算法面试最常用的语言
2. **C++为辅**：提供C++实现作为补充，帮助理解不同语言的实现差异
3. **代码质量**：
   - 确保代码可以直接运行
   - 变量命名清晰易懂
   - 注释详细说明关键步骤
   - 符合语言规范和最佳实践

## 使用示例

**用户操作**：
1. 将题目文件（如 `1.两数之和.md`）放入 `LeetCoding/` 目录
2. 告诉你："请使用/激活leetcode-analyze-skill这个skill给出这道题目的解析/详细分析这道题目"

**Skill执行**：
1. 读取题目文件内容
2. 按照5个阶段进行系统化分析
3. 生成完整的分析文档 `1-TwoSum-思路分析.md`
4. 保存到 `LeetCoding/` 目录

## 教学原则

1. **渐进式讲解**：从简单到复杂，从暴力到优化
2. **思维过程**：不仅给出答案，更要展示如何从小白一样一步一步按逻辑进行的思考过程
3. **知识关联**：将题目与算法模式、数据结构知识点关联
4. **实践导向**：提供可运行的代码和实用的调试技巧
5. **举一反三**：推荐相似题目，帮助巩固知识点
